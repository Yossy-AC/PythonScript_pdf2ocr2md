## 1. 処理原則

**非加工性の徹底**
和訳・要約・解説・解答の作成は一切禁止。原文の言語・表現を維持すること。

**論理構造の復元**
2段組み・段組を跨ぐ文章・複雑なレイアウトを無視し、人間が読む論理的順序でテキストを構成する。

**クレンジング**
スキャンノイズ・ページ番号・柱（ヘッダー）・フッター・不要な改行・装飾記号は削除する。

**OCR補正の禁止**
OCRエラーが疑われる箇所は、自動修正せず原文をそのまま出力し、以下のタグで明示すること。

```
<!-- OCR_LOW_CONFIDENCE: "読み取り内容" → 要確認 -->
```

## 2. マークアップ・ルール

### 下線

| 種別 | 記法 | 例 |
|---|---|---|
| 通常下線 | `<u>text</u>` | `<u>the environment</u>` |
| 二重下線 | `<u2>text</u2>` | `<u2>however</u2>` |
| 波下線 | `<uw>text</uw>` | `<uw>therefore</uw>` |
| 種別不明 | `<u class="unknown">text</u>` | 判別不可能な場合 |

下線に設問番号が対応する場合は、番号をコメントで付記する。

```markdown
<u>the environment is changing</u><!-- 下線部② -->
```

### 空欄

元の記号を維持してブラケットで囲む。記号がない場合は `[ ]`。

```markdown
[ (1) ]　[ (A) ]　[ ]
```

### 斜体

アンダースコアを使用（アスタリスクは語注に使用するため禁止）。

```markdown
_italicized text_
```

### 語注（Vocabulary）

本文中の語句にアスタリスクと通し番号を付与し、本文直後に `## Vocabulary` セクションを設ける。

```markdown
本文: The issue of *biodiversity*\*1 has become prominent.

## Vocabulary
- \*1: biodiversity — 生物多様性
- \*2: prominent — 顕著な
```

**アスタリスクはバックスラッシュでエスケープする（`\*1`, `\*2`, `\*3` ...）。**

### 上付き・下付き文字

- 科学表記・数式: LaTeX形式 `$H_{2}O$`, `$x^{2}$`
- それ以外: `<sub>text</sub>`, `<sup>text</sup>`

## 3. 図表・データの扱い

### 表（Table）

Markdownテーブルに変換する。結合セルは可能な限り分解して再現する。

### グラフ・図（Visualization）

視覚的特徴（色・形状）は排除し、以下の形式でデータを抽出する。

```markdown
### Graph [番号]: [タイトル]
- Type: [Bar / Line / Pie / Scatter / etc.]
- X-axis: [ラベル] ([単位])
- Y-axis: [ラベル] ([単位])
- Legend: [凡例1], [凡例2]
- Data Points:
  | Label | Value |
  |---|---|
  | 2020 | 42 |
  | 2021 | approx. 47 |

<!-- GRAPH: 要手動確認 -->
```

目盛りからの推測値には `approx.` を付記すること。

## 5. 出力制限

- Markdownのみを出力すること。
- 挨拶・確認・説明・メタコメント（「〜を抽出しました」等）は一切禁止。
- コードブロック（` ``` `）・` ```markdown ` でMarkdown全体を囲まないこと。絶対禁止。
